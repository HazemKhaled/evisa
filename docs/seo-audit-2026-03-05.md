# SEO Audit Report

Date: 2026-03-05
Scope: `apps/website` (code-based audit)

## Executive Summary

The website has a solid multilingual SEO foundation (localized metadata, JSON-LD usage, and App Router metadata APIs), but there are several high-impact technical issues affecting crawlability and indexation. The top priorities are fixing sitemap/robots alignment, correcting destination blog URL mismatches, and improving sitemap completeness for dynamic routes.

## Findings

### 1) `robots.txt` points to a likely missing sitemap URL (High)

- Evidence: `apps/website/src/app/robots.ts:15` uses `${env.baseUrl}/sitemap-index.xml`.
- Issue: No corresponding `sitemap-index.xml` route was found in `apps/website/src/app`.
- Impact: Crawlers may miss the intended sitemap index, reducing discovery and indexation depth.
- Fix: Point `robots` to an existing sitemap endpoint, or implement the missing sitemap-index route and keep naming consistent (`sitemap-index.xml` vs `sitemap_index.xml`).

### 2) Primary sitemap only includes static pages; dynamic SEO pages are missing (High)

- Evidence: `apps/website/src/app/sitemap.ts:12` defines only static pages (`""`, `"contact"`, `"about"`, `"terms"`, `"privacy"`, `"d"`).
- Issue: Destination pages, destination blog pages, blog posts, and tag pages are not included in the main discovery flow.
- Impact: Large portions of indexable content may be crawled slower or not consistently discovered.
- Fix: Build a true sitemap index and split into logical sitemaps (static, destinations, blog posts, tags, possibly per-locale).

### 3) Destination blog route mismatch creates broken internal URLs/canonical targets (High)

- Evidence: `apps/website/src/app/[locale]/blog/page.tsx:97` and `apps/website/src/app/[locale]/blog/page.tsx:142` build `/[locale]/blog/d/[destination]`.
- Evidence (actual implemented route): `apps/website/src/app/[locale]/d/[destination]/blog/page.tsx`.
- Impact: Broken links harm crawlability, user experience, and internal link equity.
- Fix: Update destination blog URL generation in blog list/search/pagination to `/${locale}/d/${destination}/blog`.

### 4) Blog sitemap hreflang alternates logic is effectively self-locale only (Medium-High)

- Evidence: `apps/website/src/app/[locale]/blog/sitemap.ts:63` uses `lang === locale ? p.slug === post.slug : false`.
- Issue: Alternates for other locales will usually never be emitted.
- Impact: Weak international SEO signals and less reliable language clustering in search.
- Fix: Map equivalent posts across locales via a stable cross-locale key (e.g., shared `postId`), not slug equality scoped to same locale only.

### 5) Tag sitemap URLs are not URL-encoded (Medium)

- Evidence: `apps/website/src/app/[locale]/blog/sitemap.ts:99` and `apps/website/src/app/[locale]/blog/sitemap.ts:104` use raw `${tag}`.
- Impact: Invalid/fragile URLs for tags with spaces/special characters; crawl inefficiency.
- Fix: Use `encodeURIComponent(tag)` in tag sitemap entries and alternates.

### 6) Inconsistent canonical strategy (Medium)

- Evidence: `apps/website/src/app/[locale]/d/page.tsx:49` uses relative canonical `/${locale}/d`, while other pages use absolute canonical helpers.
- Impact: Inconsistency can create ambiguity in canonical processing across environments.
- Fix: Standardize all canonicals via `generateAlternatesMetadata(...)` with absolute URLs and add `x-default` where appropriate.

### 7) Duplicate JSON-LD script IDs across pages (Low-Medium)

- Evidence: `apps/website/src/components/json-ld.tsx:14` hardcodes `id="json-ld"` for every JSON-LD block.
- Impact: Invalid duplicate IDs in DOM; can complicate debugging/parsing behavior.
- Fix: Generate unique IDs per block (or prefer `MultipleJsonLd` consistently with indexed IDs).

### 8) Image optimization is globally disabled (Medium, CWV risk)

- Evidence: `apps/website/next.config.ts:9` sets `unoptimized: true`.
- Impact: Potentially larger payloads and worse LCP on image-heavy pages.
- Fix: Re-enable optimization path compatible with your Cloudflare/OpenNext setup, then validate LCP/INP/CLS in production.

### 9) Localized metadata fallback is hardcoded English in destination 404 metadata (Low)

- Evidence: `apps/website/src/app/[locale]/d/[destination]/page.tsx:60` title is `"Destination Not Found"`.
- Impact: Weaker localization consistency in SERPs for non-English locales.
- Fix: Move this fallback title/description into locale translations.

## Assumptions and Coverage Notes

- This is a code audit, not a live crawl.
- Search Console, production crawl logs, and field Core Web Vitals data were not analyzed.
- Structured data is present in code, but rich-result eligibility should be validated with Google Rich Results Test for key templates.

## Prioritized Enhancement Plan

### Week 1: Indexation-Critical Fixes

1. Fix robots sitemap target in `apps/website/src/app/robots.ts`.
2. Implement a sitemap index route and submit it in Search Console.
3. Fix destination blog path generation in `apps/website/src/app/[locale]/blog/page.tsx`.
4. Encode tag URLs in blog sitemap.

### Week 2: Sitemap Architecture

1. Split sitemaps by type and locale: static, destinations, blog posts, tags.
2. Add destination-level URLs from DB-backed routes.
3. Add reliable `lastModified` from content timestamps instead of broad monthly defaults.

### Week 3: International SEO Hardening

1. Introduce robust cross-locale post mapping for hreflang alternates.
2. Standardize canonical generation everywhere with one helper.
3. Add `x-default` hreflang for global/fallback behavior.

### Week 4: Rich Results and On-Page Improvements

1. Ensure unique JSON-LD script IDs and validate all templates in Rich Results Test.
2. Add/expand schema on key templates (destination list, destination details, blog index, legal pages as appropriate).
3. Ensure page-level metadata is unique for filtered/paginated states where indexable.

### Week 5: Performance and Quality Signals

1. Re-enable image optimization pipeline for production.
2. Run PageSpeed + CWV checks on home, destinations list, top destination page, and blog post page.
3. Optimize top LCP elements (hero images, font loading, third-party scripts).

### Week 6: Measurement Loop

1. Track impressions/clicks/index coverage for `/d/*`, `/blog/*`, and locale segments.
2. Monitor crawl stats and excluded URLs.
3. Iterate based on query-to-page mapping and cannibalization checks.
